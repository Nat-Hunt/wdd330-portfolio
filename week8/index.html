Chapter 8: Transforms and Transitions

Transforms

The transform property is used to translate, rotate, scale, and skew any element. CSS3 provides more nuanced control of previously existing CSS features. Multiple transform functions (separated by spaces) can be applied at once, they will be applied in the order they're provided. All modern browsers (starting with IE10 and Android 3) support 3D transformations too, but only 2D transformations are covered here. Note: transforms do not work on inline elements, they must be block elements

Translation

Translation functions are used to move an element left, rught, up or down. These functions allow you to move an elemnt without impacting the flow of the document. They can only be moved relative to their current position, not their parent/ancestor.

Syntax:
transform: translate(45px, -45px);

If only vertical or horizontal movement is required, then use translatex or translatey respectivley.

transform: translateX(45px); transform: translateY(45px);
Note: Transforms and Older Browsers

Include the following when working with IE9, Android up to 4.4.3, iOS8, and Blackbery 10:
-webkit-transform: translate(); /* iOS8, Android 4.4..3, BB10 */
-ms-transform: translate() /* IE9 only */

Scaling

scale(x,y) scales an element horizontally then vertically. If only one value is provided, then it'll be used for the x and y values while maintaining it's original aspect ratio. You can also use scaleX(x) or scaleY(y) to only scale the specific dimensions. A scaled element will grow or shrink with respect to its center. This can be changed using the transform-origin property.

Syntax:
transform: scale(1.5);

Rotation

rotate() rotates an element around the point of origin according to the specified angle value. The default point of origin is the element's center. Angles are declared in degrees, positive creates clockwise movement, negative creates counterclockwise movement. Values can also be provided in grads, radians, or turns.

Syntax:
transform: rotate(10deg);

Skew

skew(x,y) creates a skew along the x and y axes. If only one parameter is provided, the skew will only happen along the x axis. Also has the skewX() and skewY() options

Syntax:
transform: skew(15deg, 4deg);

Changing the Origin of the Transform

The transform-origin property can be used to control the point form which transforms are applied to an element. It has the same syntax as the background-position property.

Syntax:
transform-origin: 0 0;

Note: Choose Your Ordering Carefully

The order of the transform functions is extremely important. For example, if you rotate first then translate, the translate directions will be on the rotated axis, not the original axis.

Support for Internet Explorer 8 and Earlier

Before IE 9, transforms are unsupported, but mimicable using filters and relative positioning

Transitions

Transitions generally require less client-side processing and are often a better choice than JavaScript animations. They do not function on any Internet Explorer earlier than 10, but they are UI enhancements and fail accessibly, so that doesn't matter as much.
Transitions allow for change over time to an CSS property (essentially, they're simple animations). These are the steps to create a simple transition using only CSS

  1. Declare the original state of the element
  2. Declare the final state of the transitioned element
  3. Include teh transition functions in the default style declaration using the translation properties: transition-property, transition-duration, transition-timing-function, and transition-delay

the transition is declared in the default (originating) state. Also, the -webkit- vendor prefix is necessary for older mobile devices (iOS6.1, BlackBerry10, Android4.3, and UC Browser for Android)

transition-property

This defines the CSS properties of the element that will be transitioned. all for the properties is the default. Any changing property with a valid midpoint can be transitioned. Color and size can be transitioned, but something like border-style changing from solid to dashed can't since there is no midpoint. A full list of properties that can be transitioned can be found here: https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_animated_properties
Multiple properties can be added to transition-property separated by commas. Or you can use all so that every supported property is animated. Without specifying the duration of the transition, nothing will happen.

Syntax:
transition-property: transform, skew, color;

Note: Supporting Older Browsers

Syntax:
-webkit-transition-property: -webkit-transform;
transition-property: transform;

The transition-duration Property

transition-duration determines how long the transition will take. Can be specified in seconds (s) or milliseconds (ms). 200ms is usually the best duration for a transition, going slower tends to make the website seem slow (even if it isn't), going faster may not be noticed.

Syntax:
transition-duration: 0.2s;

The transition-timing-function Property

transition-timing-function allows for controlling the pace of the transition in great detail (start slow then become fast, start fast then become slow, etc.). These are the most common timing function key terms:

  • ease => slow start, then speeds up and ends slowly
  • linear => a transition that animates at a constant speed
  • ease-in => a transition that's slow to start, gains speed, then abruptly stops
  • ease-out => starts at full speed, gradually slows
  • ease-in-out => similar to ease, but accelerates quicker in the beginning

Usually, it's helpful to make the duration long when testing these out. You can create your own timing functions by defining yor own cubic-bezier function (syntax: cubic-bezier(0, 0, 1, 1) [this creates a linear timing function]). This is an advanced calculus concept. Some useful resources for this:

  • https://cubic-bezier.com/
  • http://estelle.github.io/animation/files/cubicbezierprint.html

The steps(n, end) function can be used to step through the animation with a number of equidistant steps equal to n with either the first step occuring at the start (replace end with start), or the last step occuring at the end (leave end).

The transition-delay Property

transition-delay is used to delay the start of a transition. Takes either seconds (s) or milliseconds (ms). Useful if you don't want the animation to fire when the user is inadvertantly triggering the new state.

Tip: Negative Delays

Negative delays that're less than the duration of the entire transition will fire the transition immediately, but start it partway through the animation. Example: a dealy of -500ms on a 2s transition will start at .5 seconds and play for 1.5 seconds.

The transition Shorthand Property

Example: .ad-ad2 h1 span {
transition-property: transform;
transition-duration: 0.2s;
transition-timing-function: ease-out;
transition-delay: 50ms; }
can be changed to:
.ad-ad2 h1 span {
transition: transform 0.2s ease-out 50ms; }

Multiple Transitions

The transition property can be used for multiple transitions in one call. This is done by using separating each transition by a comma. Each subsequent transition-property declaration will need a comma separated property for each transitioned element. Example:
transition-property: transform, color;
transition-duration: 0.2s, 0.1s;
transition-timing-function: ease-out, linear;
transition-delay: 50ms;
Or if using shorthand:
transition: transform 0.2s ease-out 50ms, color 0.1s linear 50ms; If you leave the extra declarations out, then the same transitions will be applied to each element.

Animations

CSS animations allow for control of each step of animation (unlike transitions) by using keyframes.

Keyframes

First, create a named animation, then attach it to an element in its property declaration block. The @keyframes rule is used for IE10+ and FF16+. Include the @-webkit-keyframes for all WebKit implementations then a name used as an ID for that animation. Then each keyframe can be specified.

Syntax: 

@keyframes moveRight { 
    from {
    transform: translateX(-50%);
    }
    to {
    transform: translateX(50%);
    }
}

@keyframes appearDisappear { 
    0%, 100% {
    opacity: 0;
    }
    20%, 80% {
    opacity: 1;
    }
}

@keyframes bgMove {
    100% {
    background-position: 120% 0;
    }
}

Animation Properties

  • animation-name: appearDisappear; => used to attach an animation defined in the @keyframes DO NOT USE QUOTES AROUND THE NAME. This is the only required animation property.
  • animation-duration: 300ms; => used to define the length of time (seconds/milliseconds) it takes to complete an animation once from 0% to 100%. This should be required since without declaring it, it defaults to 0s

animation-timing-function

This functions the same as the transition-timing-function

animation-iteration-count

animation-iteration-count allows you to define how many times an animation will play. Typically an integer, but can use decimal points (ending the animation partilly through the iteration), or infinite which prevents it from stoping. Defaults to 1.

animation-direction

Can be used to adjust the normal flow of the animation. Normal flow is 0% => 100%. reverse makes it flow backwords from 100% => 0%. alternate makes it alternate from normal to reverse every other iteration. The alternate-reverse functions the same as alternate, but opposite.

When animations are reversed, timing functions are also reversed.

animation-delay

animation-delay determines how many seconds/milliseconds to wait before starting the animation

animation-fill-mode

animation-fill-mode determines what happens before the animation begins and what happens when the last iteration finishes. By default, there is no effect. Avaiable values:

  • none => nothing happens. It returns to normal
  • forwards => holds on the ending animation after it has played
  • backwards => holds on the starting animation until triggered to play.
  • both => holds on the starting befire and ending frame after triggering.

animation-play-state property

animation-play-state defines whether or not an animation is running or paused. Paused displays a static still of the animation until resumed.

The Shorthand animation Property

Example:

.verbose {
    animation-name: appearDisappear; 
    animation-duration: 300ms; 
    animation-timing-function: ease-in;
    animation-iteration-count: 1;
    animation-direction: alternate;
    animation-delay: 5s;
    animation-fill-mode: backwards;
    animation-play-state: running;
}

/* shorthand */
.concise {
    animation: 300ms ease-in alternate 5s backwards appearDisappear;
}
Warning: Be Careful with Naming

Avoid using key-terms in animation naming since the browser will mistake the first use of a key-term as the term and will not interpret it as a name.

An example of multiple animations using shorthand:

.target {
    animation: 
    animationOne 300ms ease-in backwards, 
    animationTwo 600ms ease-out 1s forwards;
}
