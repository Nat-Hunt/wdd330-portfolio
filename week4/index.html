A Searching Example

  • searchingExample.html
  • the action attribute is used to designate where a form will submit it's information to be processed
  • Accessing Form Elements
    • Using a numerical index: const form = document.forms[0];
    • Using the form tagname: const form = document.getElementsByTagname('form')[0];
    • Using the name attribute: const form = document.forms.search;
      Careful with this, if the name of the form shares its name with a method of the document.forms object, you'll reference that instead of the form
    • Using sqaure bracket notation: const form = document.forms['search'];
      This avoids the potential issues with using the name attribute and handles cases where the name of the form's name contains any invalid characters (like spaces or dashes)
    • const [input,button] = form.elements
      This returns a collection of all elements contained in the form
    • const input = form.searchInput
      This returns the specific element using the 'name' attribute as a property
    • const input = form['searchInput']
      Using square bracket notation is also useful
  • Form Properties and Methods
    • form.submit() submits the form automatically and skips the submit event
    • A form can be submitted manually by using a button or input element with a type attribute of submit, or an input element with a type attribute of image:
    • form.reset() resets all form fields back to their starting values. This can also be done using a button with a type attribute of reset. Reset buttons are considered poor for usability since the user may accidentaly erase all of their progress
  • Form Events
    • The focus event is triggered when an element is focused on
    • The blur event is triggered by moving focus away from the element
    • The change event is triggered when focus is removed from an element after a change ahs been made.
  • Submitting a Form
    • The submit event happens when the form is submitted
    • Using an event listener, JavaScript can interrupt a submit event. JavaScript can also stop the form submission entirely by using the preventDefault() method
  • The value property is used to retrieve and change text inside of the input element
  • Form Controls
    • input: Used for text, passwords, check boxes, radio buttons, and file uploads. Some attributes of the input tag are:
      • autofocus: will give focus on a page load
      • placeholder: will temporarily fill the field with text until the user enters text
      • maxlength: will limit the number of characters that can be entered
    • select: Used for drop-down lists of options
    • textarea: used for longer text entry
    • button: Used for submitting and resetting forms

Input Fields

  • Using the type="password" conceals the characters as they are entered
  • Checkboxes are used for selecting multiple options from a set.
  • Radio buttons are used for for selecting a single option from a set. A group of radio buttons is defined by the name attribute.
  • Hidden input fields (type="hidden") are not displayed by the browser and are used to pass along settings or information that the user has already provided. Hidden fields ARE NOT secret and should not contain sensitive information
  • file input fields (type="file") are typically furnished with a browse button by the browser
  • Other input field types:
    • number: only accepts number
    • tel: only accepts valid phone-numbers
    • color: only accepts valid color codes
    • email: only accepts valid email addresses
  • Select drop-down lists are used for selecting one or more options from a list of values. NOTE: if you want to select mulitple, the multiple attribute is required
  • Text areas are used for inputting large amounts of text
  • Buttons have three types:
    1. submit: The default type. Used to submit forms
    2. reset: Used to reset all fields in a form. This is not considered good-practice as it makes for poor usability
    3. button: Used to make a generic button with no default behavior. Useful for attaching event listeners with JavaScript

Form Validation

  • It is advisable to perform validation on both the client and server side of a webpage
  • HTML5 has a built-in validation API which isn't fully supported from all browsers, but is useful. More info can be found here
  • Most form validation can be done using JavaScript.

Disabling the Submit Button

  • Can be done by adding the disabled attribute to the submit button.
  • Can also be done programmatically. This function disables a button if an input field is empty:

Chapter 12: Object-Oriented Programming in Javascript

Object-Oriented Programming

  • Object-Oriented Programming (OOP) consists of a few main concepts: Encapsulation, Polymorphism, Inheritance, and Classes
  • Encapsulation: Hiding the inner workings from the user. Only exposing what is needed to run the object
  • Polymorphism: The same process can be sued for different objects. In other words, many objects can share the same method and override their shared method(s) with a specific implementation
  • Inheritance: Making a new object using the same properties and methods of another object and adjusting them.
  • Classes: used to define a blueprint for an object. Objects are then created using that blueprint. JavaScript didn't have classes prior to ES6. It uses the prototypal inheritance model.

Libraries

  • Library - Definition: a piece of code that provides methods that make common tasks easier to achieve
  • jQuery
    • The most popular JavaScript library in use today
    • Has a plugin system that makes it easy to extend and use to build common web page elements.
    • Has its own testing library called QUnit
    • Has a lot of support for older browsers (especcialy internet explorer)
    • This book has more information on using jQuery
  • Underscore & Lodash are similar libraries in what they provide. They are used as core dependencies for many additional libraries. They add simple functionality to much of the JavaScript language.

Modular JavaScript

  • Module - Definition: a self-contained piece of code that provides functions and methods that can then be used in other files and by other modules.
  • Code in a module should have a single purpose. For example, keep functions fused for Ajax in one module. This could then be used in any project using Ajax.
  • The goal of modules is to keep code loosely coupled and therefore interchangeable.
  • Modules must list what they export to other files. Modules can export variables, functions, and objects. For example:
    export {()=>{return 2}, AnObject = {}, const foo = 'bar'};
  • The syntax for importing a module is: import {foo} from './path/to/module/bar.js'
  • You can define a default export for a module by using the keyword 'default' like this:
    export default FOO;
    NOTE: default exports can only export a single variable, function, or class. This does allow you to forgo explicitly naming the export value that you want imported.
  • To import a default value, this is the syntax: import FOO from './path/to/module/bar.js'
  • You can assign an alias to the default import
  • Node.js Modules
    • Node.js implemented modules before ES6 and use the Common JS modules notation. Currently, there has not been a way to merge ES6 module notation and Common JS modules notation.
    • This is the syntax for exporting from a file using Common JS: module.exports = foo
    • This is the syntax for importing from a file using Common JS:
      cost bar = require('./path/to/module/bar.js')

MVC Frameworks

  • MVC stands for Model-View-Controller and it is a design pattern
  • MVC seaprates an application into three components that interact with each other. The Models, the Views, and the Controllers.
  • Models: objects that implement CRUD tasks and any other associated logic and behavior. This is the part that usually interacts with the database.
  • Views: create and update the visual representation of the model. This is what the user interacts with.
  • Controllers: the link between models and views. Controllers respond to events and user input, process the information, then update the models and views.
  • list.html
  • main.js
  • MV* Frameworks make following the MVC pattern easier. 3 common ones are:
    • Aurelia: Uses ES6. Focuses on dynamic web applications with background implementation details
    • Angualar: Framework written by Google. Extends the HTML language using custom ng- attributes
    • Ember: Designed to ease the process of building large web applications. Uses common conventions to avoid needing loads of set-up code (which can be a problem if you don't follow those conventions)

Templates

  • Templating languages are used to insert dynamic data into the page. Some use HTML, others use another coding language that compiles into HTML.
  • Often, templates are used to update only a part of the page rather than the whole thing. This reduces the number of HTTP requests needed and doesn't require the large amount of resources needed to process large strings of HTML
  • The templates are usually stored in a separate file or their own script tag.

Package Managers

  • These are used to manage JavaScript modules. Information regarding installed packages in an application are stored in a package.json file.
  • Package managers simplify the process of installing, upgrading, and removing packages/modules used in a project.
  • They also help with dependency management
  • npm, Node Package Manager, is a package manager developed shortly after Node.js.

Deploying JavaScript

  • If you have more than 1 JavaScript file, then the best way to include them in your HTML file is to bundle and minify all of your javascript files together. The typical naming convention for this is 'bundle.min.js'. Put this in a script tag on the line beofre the closing body tag.
  • Code Transpilers convert code into JavaScript, allowing for browsers that may be behind-the-curve on their JavaScript implementation to service developers who want to use the latest and greatest. Babel is the most pupular transpilar for converting ES6 into older versions.
  • Minification removes unnecessary characters to reduce file size. This includes whitespace, all comments, and other surperfluous characters. Popular minifiers include:
    • YUI Compressor
    • Google's Closure
    • UglifyJs
