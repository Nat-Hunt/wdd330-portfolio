Chapter 9: The Window Object

  • JavaScript environments have a global object. All global-scope variables are properties of the global object. Functions are methods of it. For browsers, that object is the window object.

The Browser Object Model (BOM)

  • collection of properties & methods with info on the browser & device screen.
  • Can be used to check screen dimensions, which pages were visited before this page, or to make pop-up windows.
  • global variables are properties of the window object. For example:
    x = 6;
<< 6
     
window.x;
<< 6
     
window.x === x;
<< true

Browser Information

  • You can check which browser and OS is being used by accessing window.navigator.userAgent which will return a reference to the Navigator object and then obtain the userAgent information (this tells you browser and OS is being used). It returns this string on my PC:
    "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/107.0.0.0 Safari/537.36"
  • the userAgent property is deprecated from the official specification, but is still supported in all major browsers
  • window.location is an object with info about the URL of the current page
    • window.location.href returns the full URL as a string
    • window.location.protocol returns a string with the protocol used (http, https, pop2, ftp, etc.). The string always has a colon at the end.
    • window.location.host returns the domain of the current URL and the port number (omitted if default port 80 is used)
    • window.location.hostname returns the domain of the URL
    • window.location.port returns the port number. Returns and empty string if not specified explicitly in the URL
    • window.location.pathname returns the path to the current URL
    • window.location.search returns a "?" along with the query string parameters
    • window.location.hash returns a "#" followed by the fragment identifier. Returns an empty string if non-existant
    • window.location.origin returns the protocol and domain where the current page originated from. Read-only
    • window.location.reload() forces a reload of the current page. If true is passed in, will reload from the server, not the cache
    • window.location.assign() used to load another resource from a URL provided as a parameter
    • window.location.replace() function the same as assign(), but does not store the current page in session history, thus preventing the user from returning to it using the back button.
    • window.location.toString() returns a string containing the whole URL

The Browser History

  • window.history is used to get info regarding previously visited pages in the current sesion. Not the same as the new HTML5 History API More info on the History API
  • window.history.length displays number of pages visited before the current page
  • window.history.go() used to go to a specific page where 0 is the current page. Example: 
    window.history.go(1); // goes forward 1 page
window.history.go(0); // reloads the current page
window.history.go(-1); // goes back 1 page

Controlling Windows

  • window.open() opens a new window. Has three parameters: URL of the page to be opened, the title of the window, and a list of attributes. Can also be stored in a variable for ease of reference later.
  • window.close() closes a window that you have a reference to.
  • window.moveTo(X, Y) takes 2 parameters (the X and Y coordinates) and then moves the window to that part of the screen
  • window.resizeTo(width, height) changes the window's dimensions
  • These methods aren't a very good idea to use. There's almost always a better way, so figure it out

Screen Information

  • window.screen contains info about the screen the browser is displayed on.
  • window.screen.height returns the height
  • window.screen.width returns the width
  • window.screen.availWidth returns the width exluding any OS menus
  • window.screen.availHeight returns the height exluding any OS menus
  • window.screen.colorDepth returns teh color bit depth of the user's monitor. Not super useful beyond collecting user statistics
  • This object is more useful for mobile devices. Many of these methods have been used maliciously or poorly in the past. A lot of what can be accomplished with these methods can be accomplished with media queries instead.

The Document Object

  • document.write() accepts a string and prints it to the HTML page. If used on a page that is already loaded, it will completely replace it. Can be used within the HTML file inside of a script tag, but that is fairly useless. There may be a few use cases for it, but try to use something else.

  • Cookies:

    • Store files locally about the User.
    • Restricted to only be accessible by the domain that set them.
    • limited to 4kb of data with 20 cookies per domain allowed.
    • Often used for personalizing the user browsing expierience, storing preferences, keeping track of choices (shopping cart), authentication, and tracking (frowned own in recent years).
    • Being replaced by the localStorage API which allows more data to be stored
    • Useful for storing state info (is the user logged in?)
    • They are a text file with a list of name/value pairs separated by semicolons.
      Syntax:
      "name=Superman; hero=true; city=Metropolis"
    • The EU Cookie Directive requires webistes based in the EU to get permission before setting cookies. This may become a global broswer setting rather than requiring each website to ask permission.
    • document.cookie = ''; creates a cookie. Acts like a special type of string. Assigning another cookie won't overwrite the entire property, just append it to the end fo the string.
    • The value of a cookie may be changed by reassigning it to document.cookie using the same name, but a different value.
    • document.cookie allows you to read all cookies. This example breaks the cookie string into an array and then parses it for ease of reading: 
      const cookies = document.cookie.split("; ");
for (crumb of cookies) {
    const [key,value] = crumb.split("=");
    console.log(`The value of ${key} is ${value}`);
}
    • By default, cookies are deleted when a browser session ends (close the tab/window).
    • Cookies can be make persistent by adding "; expires=date" to the end of a cookie when it is set. date is a date value following the UTC String format Day, DD-Mon-YYYY HH:MM:SS GMT
    • Persistent cookies can also be made using max-age=(number of seconds), but isn't supported by Internet Explorer before version 10
    • Sensitive information shouldn't rely on cookie expiration since browsers can sometimes retain cookie information when "sesion restore" is used after a crash.
    • cookies can only be read inside teh same directory and domain as the cookie was created in. Put their path can be changed so that each page originating in the root directory has access by adding "; path=/" or "; domain=domainName" to the end of the cookie when it is created.
    • "; secure" makes teh cookie only transmittable over HTTPS
    • To delete a cookie, set it to expire in the past.

Timing Functions

  • window.setTimeout(callbackFunction, numberOfMilliseconds) returns an integer as an ID used to reference that timeout. window.clearTimeout(id) prevents the timeout from occuring.

setInterval()

  • window.setInterval(callbackFunction, numberOfMilliseconds) is similar to setTimeout(), except it repeatedly triggers the callback function everytime the given number of milliseconds passes. window.clearInterval(id) stops it.

Note: setInterval() & Timing Functions

  • When using either of these methods, ensure that the callback function does not use the this keyword since this will be bound to the global window object instead of the callback function's object.

Chapter 14: HTML5 APIs

HTML5

  • W3C plans to update HTML5 more frequently using smaller version increments.

The data- Attribute

  • The data- attribute is used to embed data in a page using custom attributes that're ignored by the browser.
  • They're private to a page and not intended for use by external service. Meant to be used by a JS script.
  • Must:
    1. start with data-
    2. only use lowercase letters, numbers, hyphens, dots, colons, or underscores
    3. include an optional string value.
    Examples:
    • data-powers = 'flight superSpeed'
    • data-rating = '5'
    • data-dropdown
    • data-user = 'DAZ'
    • data-max-length = '32'
  • Each document element has a dataset property used to access the data- it contains. For example, the previous data-powers example can be accesssed like this: 
    div id="hero" data-powers="flight superSpeed">Superman /div 
    
const superman = document.getElementById('hero');
const powers = superman.dataset.powers;
<< 'flight superSpeed'
    NOTE: hyphenated data- attribute names are replaced with camel-case. So data-max-length would be dataset.maxLength
